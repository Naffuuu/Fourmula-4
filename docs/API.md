# API Reference

Base URL: `http://localhost:8000/api/v1`. Interactive Swagger docs are auto-generated by FastAPI at `http://localhost:8000/docs` once the backend is running ‚Äî this file is a quick-reference companion, not a replacement for it.

All error responses share one shape: `{"detail": "..."}` (or a list of field errors for 422s), regardless of endpoint ‚Äî see `main.py`'s exception handlers.

## Auth ‚Äî `/auth`

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/auth/signup` | none | Create an account. Sets refresh-token cookie, returns access token + user. |
| POST | `/auth/login` | none, rate-limited | Email/password login. Rate-limited per IP to slow brute force. |
| POST | `/auth/refresh` | refresh cookie | Rotates the refresh token, issues a new access token. Called automatically by the frontend on 401s and on hard page reloads. |
| POST | `/auth/logout` | refresh cookie | Revokes the refresh token server-side and clears the cookie. |
| GET | `/auth/me` | bearer | Current user profile. |
| POST | `/auth/forgot-password` | none | Always returns 202 regardless of whether the email exists (no account enumeration). Dev mode logs the reset link to the backend console. |
| POST | `/auth/reset-password` | none | Consumes a reset token, sets a new password. |

## OAuth ‚Äî `/oauth`

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/oauth/google` | none | Body: `{"credential": "<Google ID token from GIS>"}`. Verified server-side against Google's public keys. |
| POST | `/oauth/facebook` | none | Body: `{"credential": "<Facebook access token from FB.login()>"}`. Verified server-side via the Graph API's `debug_token`. |

## Dashboard ‚Äî `/dashboard`

| Method | Path | Auth | Description |
|---|---|---|---|
| GET | `/dashboard/summary` | bearer | Aggregated live numbers: strike count, open SOS alerts, recent complaint categories, days until next test. |

## Mission 1 ‚Äî Whistleblower ‚Äî `/complaints`

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/complaints/evidence` | bearer | Multipart file upload. Strips EXIF, stores the image, returns an `upload_id` to reference from the complaint. |
| POST | `/complaints` | bearer | Submit a complaint. Never records who submitted it beyond an anonymized bucket ref. |
| GET | `/complaints` | captains only | List all complaints for triage (never exposes submitter identity). |
| POST | `/complaints/{id}/strike` | captains only | Issues the next strike level (capped at 3) against a complaint. |

## Mission 2 ‚Äî Seat Planner ‚Äî `/seating`

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/seating/generate` | captains only | Body: roster + grid dimensions + podium/Kuddus seat + aisle columns. Returns a full grid with per-seat `sightline_blocked` flags and an average sightline score. |

## Mission 3 ‚Äî Syllabus Negotiator ‚Äî `/syllabus`

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/syllabus` | bearer | Body: raw syllabus text + test date. Returns a summary, examinable vs. dropped topics, and a time-blocked daily study plan. |

## Mission 4 ‚Äî Tiffin Ledger ‚Äî `/ledger`

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/ledger` | bearer | Log a cash or food-item entry. Anonymous by design, same pattern as complaints. |
| GET | `/ledger` | bearer | Full ledger + derived "outrage metrics" (cash in jhalmuri packets/cricket bats, estimated caloric intake). |

## Mission 5 ‚Äî SOS Flare ‚Äî `/sos`

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/sos` | bearer | Trigger an alert; broadcasts to every connected captain socket. |
| GET | `/sos` | bearer | Last 50 alerts, for reconciling the offline queue on reconnect. |
| WS | `/sos/ws?token=<access_token>` | captains only | Live feed. Access token passed as a query param since browsers can't set custom WebSocket handshake headers. |

## Mission 6 ‚Äî Fact-Checker ‚Äî `/factcheck`

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/factcheck` | bearer | Body: `{"claim": "..."}`. Returns verdict (bool), confidence score, matched rule text, and source section. |
